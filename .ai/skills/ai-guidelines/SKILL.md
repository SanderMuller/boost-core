## How boost-core works (in package context)

This package uses `sandermuller/boost-core` (the Composer plugin):

1. **Guidelines** in `.ai/guidelines/` define project-wide coding conventions
2. **Skills** in `.ai/skills/{name}/SKILL.md` provide domain-specific instructions activated on demand
3. boost-core's `SyncEngine` fans these out into the per-agent layouts (`.claude/skills/`, `.cursor/skills/`, `CLAUDE.md`, `AGENTS.md`, etc.) for all nine supported agents
4. Run via `vendor/bin/boost sync` (standalone) or `composer boost:sync` (plugin) â€” auto-runs on `composer install` / `composer update` already

## Updating Guidelines

Edit files in `.ai/guidelines/`:

- **Blade files** (`.blade.php`): Use `@verbatim` blocks around code examples
- **Markdown files** (`.md`): Standard markdown format

## Creating/Updating Skills

Skills live in `.ai/skills/{skill-name}/SKILL.md` with frontmatter:

```markdown
---
name: skill-name
description: When this skill activates. Include trigger words and scenarios.
---

# Skill Title

## When to use this skill

- Scenario 1
- Scenario 2

## Instructions

Detailed instructions for the skill...
```

### Skill Naming

- Use kebab-case for directory names: `my-new-skill/`
- The `name` in frontmatter should match the directory name
- Write clear `description` with activation triggers

## Regenerating Generated Files

Auto-sync runs on every `composer install` / `composer update` via the plugin's `onPostAutoloadDump` hook. To trigger manually after editing `.ai/`:

```bash
vendor/bin/boost sync       # standalone
composer boost:sync         # via plugin
composer sync-ai            # alias defined in composer.json
```

This fans `.ai/` changes into all configured agent locations:

| Generated File/Directory          | Purpose                                  |
|-----------------------------------|------------------------------------------|
| `CLAUDE.md`                       | Main context file for Claude             |
| `AGENTS.md`                       | Main context file for Codex / OpenCode / Amp / Cursor / Junie / Kiro |
| `GEMINI.md`                       | Main context file for Gemini             |
| `.claude/skills/<name>/SKILL.md`  | Skills loaded by Claude Code             |
| `.cursor/skills/<name>/SKILL.md`  | Skills for Cursor                        |
| `.github/skills/<name>/SKILL.md`  | Skills for GitHub Copilot                |
| `.github/copilot-instructions.md` | Instructions for GitHub Copilot          |
| `.{amp,gemini,junie,kiro,opencode,agents}/skills/<name>/SKILL.md` | Skills for remaining six agents |

Generated files are gitignored under the managed block in `.gitignore` â€” only `.ai/` sources go into git.
